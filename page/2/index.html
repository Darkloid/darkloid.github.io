https://www.win-rar.com/fileadmin/winrar-versions/sc/sc20231013/wrr/winrar-x64-624sc.exe

众所周知，WinRAR商业版是比较良心的版本，但自简体中文5.20版本之后，WinRAR就彻底变味了。变成了注册后也弹广告的“恶心”软件。好在有商业版继续良心发布，但这商业版对官方来说是“不可见光的”，也就是说没那么容易让你得到。今天我就来分析下各个版本的商业版网址特点。

官方阻止下载商业版第一招：中文版网站换成免费版，商业版仅在外国网站才可下载的到。

5.01 5.10 5.20 5.21 5.30 5.31用以下网址代入
32位：https://www.rarlab.com/rar/wrar***sc.exe
64位： https://www.rarlab.com/rar/winrar-x64-***sc.exe
***：代表版本号（3个数字）
比如说，我要5.20版本的WinRAR商业版，那么代入公式就是：
32位：https://www.rarlab.com/rar/wrar520sc.exe
64位： https://www.rarlab.com/rar/winrar-x64-520sc.exe

官方阻止下载商业版第二招：替换掉外国网站的商业版，改成他的免费版。商业版获取渠道更少。

从5.40开始，英文版官网（rarlab）再也没有简体中文商业版的下载地址了，取而代之的是恶心的免费版。 德国官网（win-rar）封了大部分渠道。只剩下一条道可获得商业版。从这里进入了“猜日期”的时代。

5.40 5.50 5.60 5.61 5.70 5.71 （还有5.70b2版）用以下网址代入
32位：https://www.win-rar.com/fileadmin/winrar-versions/sc********/wrr/wrar@@@sc.exe
64位：https://www.win-rar.com/fileadmin/winrar-versions/sc********/wrr/winrar-x64-@@@sc.exe

********：8位数的日期（纯数字）@@@：版本号（3位数）
版本号好找，那8位数的日期在哪找呢？很简单，只要去http://www.winrar.com.cn改下版本号随便下载一个任意版本免费版的WinRAR,然后右键安装包-点属性-里面有个数字签名。这就是我们需要的日期。
比如说，我们要5.40商业版的WinRAR 那就去官网改下版本号下载下来就行了（记住，如果要旧版的话不要傻乎乎的直接点下载，一定要改好相应的版本号才能下载）

下载好后，鼠标右键安装包-点属性-点数字签名会出现下图

2016年8月18日 那就记成20160818。但这20160818是大概的日期，需要前一天，后一天逐一的去试去代入。也就是说，这个日期可能是20160820 19 18 17 16 15…甚至是更长。最后试出20160819是正确的代入日期。

那代入网址就是：
32位：https://www.win-rar.com/fileadmin/winrar-versions/sc20160819/wrr/wrar540sc.exe
64位：https://www.win-rar.com/fileadmin/winrar-versions/sc20160819/wrr/winrar-x64-540sc.exe

由于时间关系，我直接给大家各个版本试日期的结果，就不做各个版本分析日期的整个过程了。
540版：20160819 （免费版日期2016年8月18日加1天）
550版：20170830 （免费版日期2017年8月30日不变）
560版：20180711 （免费版日期2018年7月10日加1天）
561版：20181016 （免费版日期2018年10月16日不变）
570b2版：20190222 （免费版日期2019年2月23日减1天）
570版：20190304 （免费版日期2019年3月4日不变）
571版：20190509 （免费版日期2019年5月24日减15天）

注：5.71版曾经发布过5月9日数字签名的免费版，在rarlab能下载。但后期不知怎么回事，rarlab现在也改成5月24日数字签名版的了，所以日期相差巨大。

官方阻止下载商业版第三招：商业版网址公式小改动，另外故意搞出弹窗BUG，目的就是为了隐藏早期发布的数字签名日期，增加日期的计算量。

从5.80版开始，商业版的公式做出了小改动，原来的公式已经不能用了
5.80之后新版公式：
32位：https://www.win-rar.com/fileadmin/winrar-versions/sc/sc********/rrlb/wrar@@@sc.exe
64位：https://www.win-rar.com/fileadmin/winrar-versions/sc/sc********/rrlb/winrar-x64-@@@sc.exe

相对于原来的网址，多加了sc/，把wrr改成了rrlb。
********和@@@代入方法和上面一样，就不多做介绍了
580版：20191217 （免费版日期2020年1月3日减16天）
590版：20200409（免费版日期2020年4月15日减6天）

注：5.80版由于弹窗问题，经历了2019年12月17日，2019年12月26日和2020年1月3日三个版本的数字签名，所以20191217和2020年的1月3日一比日期会相差较大。12月17日和26日由于有漏洞现在不能从官方渠道下载，官方渠道只剩下修复过的1月3日数字签名版的了。

5.90版和5.80版一样也持有弹窗问题，经历了2020年4月9日版和2020年4月15日版（rarlab那里的免费版是2020年4月14日版）后期不知还会不会继续发布修复版更改日期。现在2020年4月9日版官方渠道已无法下载。只有14日和15日的可在官方渠道下载。

以上就是从旧版到新版商业版的演变过程，想要考究新旧商业版网址特点的人可以用此帖做为参考。以后如果商业版变换了新的网址公式，或者是新渠道的获取商业版的WinRAR，我后期还会补充介绍的。

Windows自动睡眠失效

powercfg /requests
查看哪些进程、服务、驱动有电源需求。列表中的程序、驱动会阻止计算机进入睡眠。需要重写（删除）电源需求，方法如下。
格式：powercfg /requestsoverride 请求者类型 请求者名称 电源需求类型(可写1到3个需求类型)
举例：powercfg /requestsoverride PROCESS "chrome.exe" system
此命令会忽略掉chrome阻止睡眠的请求，名称加引号更保险。
如果你遇到下面这种带括号很长一串的情况

对应命令是：powercfg /requestsoverride DRIVER "HDAUDIO\FUNC_01&VEN_10EC&DEV_0295&SUBSYS_103C84DA&REV_1000\4&306c1263&0&0001" SYSTEM
或：powercfg -REQUESTSOVERRIDE DRIVER "Realtek High Definition Audio" SYSTEM
如果还是不能睡眠，那就需要更新音频驱动
恢复电源需求，让其继续阻止电脑进入睡眠：powercfg /requestsoverride PROCESS chrome.exe
